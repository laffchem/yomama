Yo Mama! API Docs

Why was this project needed? Honestly it wasn't, but I had a ton of fun making it, and I really hope you enjoy using it. This API doesn't require authentication to use, but please be reasonable with usage. All requests will return a json response based on the parameters you have passed to the server.

Get Random Joke Get Joke by Category Get Joke List by Category

Random Joke

To retrieve a random joke from the server, you need to perform a get request to the following endpoint.

curl -X 'GET' \ 'http://127.0.0.1:8000/api/v1/jokes/random/' \ -H 'accept: application/json'

/api/v1/jokes/random/

The response will contain an id, joke, and category.

{ "id": 0, "joke": "string", "category": "string" }

You can then use the id in the joke by id endpoint to get a specific joke.

Joke by Category

To retrieve a random joke by category from the server, you need to perform a get request to the following endpoint with a category.

curl -X 'GET' \ 'http://127.0.0.1:8000/api/v1/jokes/category/random/' \ -H 'accept: application/json'

/api/v1/jokes/category/random/

The categories available are "fat", "stupid", "ugly", "nasty", "hairy", "bald", "old", "poor", "short", "skinny", "tall", or "like". Anything else will return a 404 not found error.

The response will contain an id, joke, and category.

{ "id": 0, "joke": "string", "category": "string" }

You can then use the id in the joke by id endpoint to get a specific joke.

Joke List by Category

To retrieve a random joke list by category from the server, you need to perform a get request to the following endpoint with a category.

curl -X 'GET' \ 'http://127.0.0.1:8000/api/v1/jokes/category/' \ -H 'accept: application/json'

/api/v1/jokes/category/

The response an array of jokes that all contain an id, joke, and category.

[{ "id": 0, "joke": "string", "category": "string" }, ... ]

You can then use any of the ids in the joke by id endpoint to get a specific joke.
